Q: The 5 Code Smell categories
A: 1. Bloaters
2. Abusers
3. Preventers
4. Dispensables
5. Couplers
<!--ID: 1744849487123-->




---

Q: Bloaters description
A: unnecessarily increase the volume of code (more code to understand and maintain)
<!--ID: 1744849487126-->




---

Q: Abusers description
A: things you could do in OO but maybe you should be a little careful of, and then not do too much.
<!--ID: 1744849487127-->




---

Q: Preventers description
A: prevent people to easily make changes in code.
<!--ID: 1744849487130-->




---

Q: Dispensables description
A: arbitrary additions in code that increase its length and don't add much value (most of these things can be done in a simpler way)
<!--ID: 1744849487132-->




---

Q: Couplers description
A: bind objects together.
<!--ID: 1744849487134-->




---

Q: 5 **Bloater** smells
A: - long function
- large class
- data clumps
- long parameter list
- primitive obsession
<!--ID: 1744849487136-->




---

Q: 6 **Abuse** smells
A: - repeated switches
- refused bequest
- alternative classes with different interfaces
- temporary field
- global data
- mutable data
<!--ID: 1744849487139-->




---

Q: 3 **Preventer** smells
A: - divergent change
- shotgun surgery
- mysterious name
<!--ID: 1744849487142-->




---

Q: 6 **Dispensable** smells
A: - lazy element
- speculative generality
- data class
- duplicated code
- loops
- comments
<!--ID: 1744849487145-->




---

Q: 4 **Coupler** smells
A: - feature envy
- insider trading
- message chains
- middle man
<!--ID: 1744849487149-->
